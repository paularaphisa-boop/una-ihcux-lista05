📌 Resumo das Heurísticas Aplicadas

O projeto de conversão de moeda (Real → Dólar) foi desenvolvido com base em princípios de usabilidade definidos por Jakob Nielsen, aplicando três heurísticas principais:

🔄 1. Visibilidade do Status do Sistema

O sistema mantém o usuário informado sobre o que está acontecendo.

✔️ No projeto:
Exibição do valor convertido imediatamente após a ação
Mensagens como “Convertendo...” ou resultado exibido na tela
Atualização dinâmica do valor

👉 Isso garante que o usuário saiba que a conversão foi realizada com sucesso.

⚠️ 2. Prevenção de Erros

O sistema evita que o usuário cometa erros antes que eles aconteçam.

✔️ No projeto:
Validação do campo (não permitir valores vazios ou inválidos)
Restrição para aceitar apenas números
Uso de try-catch para evitar falhas na execução

👉 Isso impede erros comuns e evita que o sistema quebre.

🎨 3. Estética e Design Minimalista

A interface deve ser simples, clara e sem informações desnecessárias.

✔️ No projeto:
Interface limpa (apenas campo, botão e resultado)
Uso de cores e elementos visuais simples
Foco apenas na função principal: converter moeda

👉 Isso facilita o uso e evita distrações.

🧩 Conclusão

O programa aplica boas práticas de usabilidade ao:

Informar o usuário (Status do sistema)
Evitar erros antes que aconteçam (Prevenção)
Manter uma interface simples e eficiente (Estética)

👉 Resultado: uma aplicação fácil de usar, clara e confiável.
